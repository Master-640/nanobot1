Agent 架构对比实验 Skill
1. 技能概述
本技能提供一套标准化的实验流程，用于评估和对比智能体系统中两种关键架构组件的不同实现方式：

记忆机制：原生上下文窗口（滑动窗口） vs. 向量检索增强记忆

工具系统：粗粒度通用工具（如 execute_python_code） vs. 细粒度专属工具（如 calculate_average, read_csv）

实验固定 LLM 后端（DeepSeek）和部署环境，通过自动化任务脚本收集多维度指标，最终输出对比报告，指导系统架构设计决策。

2. 适用场景
研究团队或开发者在设计智能体时，需要评估不同记忆和工具实现的优劣。

对比不同架构配置在长对话、复杂任务下的稳定性、成本和效率。

为生产环境选择最优的 Agent 架构组合提供数据支撑。

3. 实验环境要求
LLM API：DeepSeek-V2 或 DeepSeek-Chat，具备稳定访问权限。

部署环境：统一容器化环境（Docker），CPU 4核，内存 16GB，Python 3.10+。

向量数据库（如使用）：ChromaDB（内存模式）或 Qdrant，搭配 embedding 模型（如 text-embedding-3-small）。

工具执行沙箱：Python 3.10 环境，允许标准库及 pandas、numpy，限制危险文件操作。

4. 实验变量与配置
4.1 记忆机制
配置	描述
滑动窗口 (SW)	仅保留最近 8 轮对话，超出部分丢弃。
向量检索 (VR)	将每轮交互编码为向量存入向量库，每轮开始时检索与当前输入最相关的 Top-3 历史片段，并保留最近 2 轮完整对话作为上下文。
4.2 工具系统
配置	描述
粗粒度 (CG)	仅提供 execute_python_code(code: str) -> str 工具，由 LLM 自行生成并执行完整 Python 代码。
细粒度 (FG)	提供一组专用工具：read_csv、calculate_average、filter_data、detect_outliers 等，每个工具功能单一，输入输出明确。
5. 测试任务
Task 1：数据统计分析
目标：计算 CSV 中 income 的平均值和标准差，筛选 age > 30 后计算 spending_score 的平均值，以 JSON 返回。

重点评估：工具系统的准确性、Token 消耗、执行时间。

Task 2：数据清洗与转换
目标：将 date 列标准化，对 sales 列进行异常值检测（均值±2倍标准差）并替换为均值，输出清洗后的 CSV。

重点评估：工具系统的鲁棒性、代码生成质量（粗粒度）或工具调用链效率（细粒度）。

Task 4：长期偏好记忆
目标：多轮对话中，前 3 轮设定用户偏好（表格输出、保留两位小数、忽略缺失值），中间 12 轮干扰对话，最后 1 轮隐性测试偏好是否被召回。

重点评估：记忆系统的召回率、遗忘轮次（滑动窗口）。

6. 实验组设计（2×2 全因子）
组号	记忆配置	工具配置
1	滑动窗口	粗粒度工具
2	滑动窗口	细粒度工具
3	向量检索	粗粒度工具
4	向量检索	细粒度工具
每组每个任务独立运行 10 次，共 120 次实验。

7. 评估指标
指标	定义	收集方式
成功率	任务输出通过验证脚本的比例	自动化脚本判断
执行时间	从请求开始到最终输出结束的耗时（秒）	时间戳计算
Token 消耗	API 返回的 prompt_tokens + completion_tokens 总和	API 响应字段
工具调用次数	单任务中工具被调用的总次数	日志统计
工具失败次数	因参数错误、执行异常等原因失败的调用次数	日志统计
记忆召回分数	Task 4 中偏好符合程度（0-3 分）	LLM-as-Judge + 人工抽检
遗忘轮次	滑动窗口组中偏好首次丢失的对话轮次	日志分析
安全性	粗粒度工具组中是否出现危险操作	代码扫描
8. 自动化执行流程
8.1 准备阶段
生成测试数据（CSV 文件）并放置于固定路径。

实现四种配置的 Agent 封装（统一接口，可切换记忆与工具）。

编写任务验证脚本（基于预期输出进行模糊匹配或数值验证）。

编写主实验脚本，支持顺序随机化、重试机制、日志记录。

8.2 运行阶段
预热：运行一轮所有配置，不计入结果。

循环执行：对每组配置，按随机顺序执行每个任务 10 次。

日志记录：每次运行记录完整对话历史、API 调用详情、工具调用详情、时间戳、Token 数。

验证：任务完成后立即调用验证脚本，标记成功/失败。

8.3 数据分析
汇总各指标的平均值和标准差。

使用卡方检验比较成功率差异，ANOVA 或 Kruskal-Wallis 检验连续变量差异。

绘制柱状图、箱线图、雷达图展示对比结果。

9. 输出报告
实验结束后生成包含以下内容的报告：

实验环境与配置说明

各配置在各任务上的指标汇总表

统计检验结果（p 值）

可视化图表

结论与架构建议

10. 使用说明
10.1 调用本技能
当用户需要执行或设计此类实验时，AI 助手可加载本技能，并根据实际环境微调细节（如任务复杂度、运行次数）。

10.2 自定义扩展
可增加更多任务（如多步推理、工具组合使用）。

可替换 LLM 后端（如 GPT-4、Claude）以评估模型敏感性。

可增加记忆机制的变体（如摘要记忆、混合记忆）。

10.3 注意事项
确保 API 密钥安全，避免在日志中暴露。

向量检索的 embedding 模型需稳定且与任务领域匹配。

粗粒度工具需配置安全沙箱，防止恶意代码执行。
