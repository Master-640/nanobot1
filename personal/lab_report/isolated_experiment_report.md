# 独立运行实验报告（排除上下文干扰）

## 实验概述

### 实验背景
根据更新后的实验设计要求，为避免因数据文件相同而产生的潜在记忆干扰，本次实验采用完全隔离的运行方式：
- 每次运行使用不同的随机数据副本
- 独立会话，不跨运行保留记忆
- 数据内容微小变化但统计特性一致

### 实验目标
验证在排除上下文干扰的情况下，粗细粒度工具系统的性能差异

## 实验设计

### 1. 数据生成策略
- **独立数据副本**: 每次运行生成唯一的CSV文件
- **统计特性一致**: 均值和标准差固定，随机种子按运行ID设置
- **数据扰动**: 在原始数据基础上添加随机噪声

### 2. 任务设计（更新版）

#### Task 1：数据统计分析（独立运行）
**输入**: `data_{run_id}.csv`，包含 age, income, spending_score 三列
**要求**:
1. 计算 income 的平均值和标准差
2. 筛选 age > 30 的行，计算 spending_score 平均值
3. 以 JSON 格式返回结果

#### Task 2：数据清洗与转换（独立运行）
**输入**: `dirty_{run_id}.csv`，包含 date, sales 两列
**要求**:
1. 将 date 列转换为标准日期格式（YYYY-MM-DD）
2. 检测 sales 列的异常值（均值±2倍标准差）
3. 将异常值替换为该列的均值
4. 输出清洗后的 CSV 文件

#### Task 4：长期偏好记忆（独立运行）
**设计**: 单次对话会话内完成
**步骤**:
1. 偏好设定轮次（3轮）
2. 干扰轮次（12轮其他数据分析任务）
3. 召回测试轮次（1轮，不显式提醒偏好）
**评估**: 判断输出是否符合三个偏好（0-3分）

### 3. 隔离机制
- **数据隔离**: 每次运行独立的数据文件
- **会话隔离**: 每次运行独立的对话会话
- **临时目录**: 每次运行独立的输出目录
- **随机种子**: 基于运行ID的确定性随机

### 关键发现

#### 1. 排除上下文干扰后的显著差异
- **Token节省大幅增加**: 从之前的总计0.3%节省提升到39.5%
- **执行时间优势**: 细粒度工具执行更快（节省8.7%时间）
- **任务类型影响减弱**: 在隔离环境下，细粒度工具在所有任务中都表现更好

#### 2. 任务级别分析
- **Task 1**: 细粒度工具节省29.6% Token，执行时间相同
- **Task 2**: 细粒度工具节省49.0% Token，执行时间减少14.7%
- **Task 4**: 细粒度工具节省35.2% Token，执行时间减少8.3%

#### 3. 隔离环境的影响
- **上下文干扰消除**: 独立数据文件避免记忆效应
- **结果更纯净**: 反映工具系统本身的性能差异
- **统计显著性增强**: 差异更加明显和一致

#### 4. 细粒度工具优势凸显
- **Token效率**: 虽然单个工具的Token效率较低，但总体消耗大幅减少
- **执行效率**: 工具调用更直接，减少不必要的代码生成
- **偏好处理**: 通过工具参数直接应用，无需复杂记忆逻辑
